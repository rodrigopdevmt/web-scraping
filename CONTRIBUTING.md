# Contribuindo

Obrigado por considerar contribuir! 🎉

## Como contribuir

1. **Fork** o repositório
2. **Crie uma branch** (`git checkout -b feature/nova-feature`)
3. **Faça suas alterações**
4. **Rode os testes** (`pytest`)
5. **Verifique o estilo** (`ruff check .`)
6. **Commit** (`git commit -m 'Adiciona nova feature'`)
7. **Push** (`git push origin feature/nova-feature`)
8. **Abra um Pull Request**

## Padrões de código

- Siga o estilo definido no `pyproject.toml` (Ruff)
- Adicione type hints em todas as funções
- Escreva testes para novas funcionalidades
- Mantenha a compatibilidade com Python 3.10+

## Relatando issues

Use o template de issue no GitHub. Inclua:
- Passos para reproduzir o problema
- Comportamento esperado vs atual
- Versão do Python e SO

## Pull Requests

- Mantenha o escopo focado (um PR por feature/fix)
- Atualize a documentação se necessário
- Verifique se todos os testes passam
